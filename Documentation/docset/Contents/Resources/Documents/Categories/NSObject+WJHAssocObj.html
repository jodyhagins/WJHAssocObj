Associate the object with the receiver using a weak reference.

- (void)wjh_associateWeakly:(id)object withKey:(void const *)key atomically:(BOOL)atomically

Parameters

object

The object being added to the association. It will be stored as a weak reference to the object. Thus, it will behave in the same way as any other weak reference. When the original object is finally deallocated, the reference in the association will be zeroed, in effect, removing it from the association.

Fetching Associations

– wjh_associatedObjectWithKey:

Fetch the object that is associated with the given key.

- (id)wjh_associatedObjectWithKey:(void const *)key

Parameters

key

The unique key that identifies the associated object

Return Value

The associated object, or nil if there is no association

Discussion

Note: Note that getting an associated weak pointer will return a strong pointer. The associated object is still weak, but the one returned will be strong, and as such will retain the underlying object.
